## 使用说明

| 标记 | 含义 |
|---|---|
| `[ ]` | 未执行 |
| `[ √ ]` | 通过 |
| `[ × ]` | 不通过，需要记录缺陷 |
| `[ 跳过 ]` | 当前环境或数据不满足，需注明原因 |

优先级：

| 优先级 | 含义 |
|---|---|
| P0 | 阻断主流程、权限、安全、数据一致性 |
| P1 | 高频业务路径或重要回归 |
| P2 | 低频能力、展示细节、边界体验 |

建议执行顺序：

1. 先跑自动化：前端单测、前端构建、后端模块测试。
2. 再跑接口级冒烟：登录、权限、预约、通知、AI 助手、管理员接口。
3. 最后跑真实浏览器：关键业务端到端、弹窗、卡片、统计图、响应式。

常用命令：

```powershell
# 前端
cd E:\project\meeting-room\frontend
pnpm test -- run
pnpm build:staging

# 后端
cd E:\project\meeting-room\backend
mvn -pl meeting-room-server test

# 本地启动
cd E:\project\meeting-room
.\start-dev.bat
```

## 19. 当前特别关注风险

| 风险 | 测试关注点 |
|---|---|
| 通知发布入口迁移 | 不再验收 `/admin/notifications` 独立菜单，重点验收顶部铃铛 admin 发布弹窗 |
| 紧急会议入口位置 | 只在管理端预约审核页验收创建入口；日历页不应出现紧急会议入口 |
| AI 助手相对日期 | 手工测试时按执行当天重新理解“今天/明天/下周”，不要直接套旧日期结果 |
| AI 评价链路 | 需要准备“已结束且未评价”的预约夹具，否则会得到不能评价的正确业务拒绝 |
| 管理端删除/重复编码 | 重点检查错误提示是否只出现一次，且弹窗位置正常 |
| `PENDING` 与 `ACTIVE` 占用差异 | 创建待审不算正式占用；审核通过和紧急会议确认必须重校验冲突 |
| 设备启停与绑定 | 禁用设备、设备不足、设备绑定数量会影响推荐和创建，应交叉验证 |
| 权限边界 | 前端隐藏不等于安全，所有 `/admin/*` 和他人预约操作必须后端拒绝 |
